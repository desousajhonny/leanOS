# Test Planning

## Purpose

Plan validation for implementation work without storing procedural test instructions as loose area files.

## Inputs

- Implementation scope
- Acceptance criteria
- Changed behavior
- Known risks

## Process

1. Identify changed behavior
2. Choose automated and manual validation
3. Map tests to acceptance criteria
4. Identify risky gaps
5. Summarize validation readiness

## Output

- Test strategy
- Validation gaps
- Manual checks
- Next action

## Files to Update

- Update `../implementation-notes.md` or PR notes if the workspace needs a persistent test decision.

Start from `../README.md`, choose a role in `../roles/`, load required skills in `../skills/`, then use this playbook.
