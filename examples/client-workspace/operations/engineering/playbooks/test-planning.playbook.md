# Test Planning

## Purpose

Plan validation for implementation work without storing procedural test instructions as loose area files.

## Inputs

- Implementation scope
- PRD
- Acceptance criteria
- Changed behavior
- Known risks
- Testing strategy
- Skill: write-tests

## Process

1. Read `knowledge/testing-strategy.md`
2. Identify changed behavior
3. Use `skills/write-tests.skill.md` to choose automated and manual validation
4. Map tests to acceptance criteria
5. Identify risky gaps
6. Summarize validation readiness

## Output

- Test strategy
- Validation gaps
- Manual checks
- Next action

## Files to Update

- Update `../knowledge/implementation-notes.md` or PR notes if the workspace needs a persistent test decision.

## Navigation

Start from `../AGENT.md`, choose a role in `../roles/`, load required skills in `../skills/`, then use this playbook.
