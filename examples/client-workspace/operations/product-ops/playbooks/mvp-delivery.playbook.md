# MVP Delivery

## Purpose

Turn product strategy into executable MVP scope.

## Inputs

- Product brief
- Problem
- ICP
- Value proposition
- Roadmap or current cycle when available
- Existing MVP scope
- Existing PRD when available

## Process

1. Read Product Ops AGENT and choose the Product Owner role
2. Read product strategy and existing MVP knowledge
3. Define the smallest coherent MVP scope
4. Write or refine the MVP PRD
5. Write or refine user stories
6. Define acceptance criteria
7. Confirm non-goals
8. Identify Design, Security, Engineering or DevOps dependencies
9. Propose file updates and wait for confirmation before writing

## Output

- MVP scope proposal
- PRD proposal
- User stories
- Acceptance criteria
- Non-goals
- Dependencies
- Open questions

## Files to Update

- Update `../mvp/scope.md`, `../mvp/prd.md`, `../mvp/user-stories.md`, `../mvp/acceptance-criteria.md` and `../mvp/non-goals.md` only after explicit confirmation.
- Update `../knowledge/overview.md` or `../knowledge/delivery-context.md` when delivery context changes.
