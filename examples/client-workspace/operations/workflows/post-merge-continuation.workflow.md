# Post Merge Continuation Workflow

## Purpose

Continue delivery after a founder confirms a merge.

## Required Areas

- core
- engineering

## Availability

All required areas are active in this department.


## Sequence

1. Confirm merge evidence or founder confirmation
2. Record relevant implementation notes
3. Identify learning or roadmap impact if any
4. Load the next issue
5. Restart issue delivery

## Navigation

Use Operations area READMEs for each step to preserve area-first ownership.
