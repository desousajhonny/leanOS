# Configure Environments

## Purpose

Plan environment boundaries and configuration without inventing project-specific infrastructure.

## Inputs

- Product stage
- Runtime requirements
- Secrets or integration needs
- Deployment target

## Process

1. Identify required environments
2. Define environment responsibilities
3. List configuration needs
4. Identify secrets and access boundaries
5. Capture open questions

## Output

- Environment plan
- Configuration needs
- Access risks
- Open questions

## Files to Update

- Update relevant DevOps notes or environment records if the workspace has them.
