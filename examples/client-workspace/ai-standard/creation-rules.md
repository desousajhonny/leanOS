# Creation Rules

- `AGENT.md` is the operating owner for its level.
- `README.md` is the directory map and explanation.
- Area `AGENT.md` files are optional and should exist only when an area needs a lead router before specialist roles.
- Create roles, skills and playbooks inside the correct area.
- Do not place roles, skills or playbooks directly under root departments.
- Every area asset must reference its area owner, local README and source-of-truth files.
- Keep LeanOS runtime files in `.leanos/` small and route-focused.
