# Create Screen Spec Skill

## Purpose

Define screen purpose, content, states and interactions.

## Inputs

- Active department README
- Active role file
- Relevant source-of-truth files

## Process

1. Confirm the active role.
2. Read only the required source-of-truth files.
3. Produce the smallest useful output.
4. Point to the next playbook or command.

## Expected Output

- Current context
- Key decision or recommendation
- Files to update
- Suggested next action

## Navigation

Return to `../README.md` or continue to a playbook in `../playbooks/`.

## Agent Notes

This skill belongs to the Design department. Do not use it as a global skill without routing through the department first.
