# Design Department

## Purpose

Own UX flows, screen specs, usability states and MVP design decisions.

## When to Use

Use this department when the user asks to:

- map user flows
- define screens
- design onboarding
- reason about usability
- document UX states

## Choose a Role

For the user asks about screens, flows, onboarding, UI or usability, use:

`roles/ux-lead.role.md`

## Common Paths

- UX request: role `roles/ux-lead.role.md` -> skill `skills/map-user-flow.skill.md` -> playbook `playbooks/mvp-ux-flow.playbook.md`.
- Screen request: role `roles/ux-lead.role.md` -> skill `skills/create-screen-spec.skill.md`.
