# MVP UX Flow Playbook

## Purpose

Create a usable flow for the first validation cycle.

## When to Use

Use this playbook when the active Design role needs a sequence of execution.

## Sequence

1. Read ICP and MVP scope
2. Map primary flow
3. Specify screens
4. Define UX states
5. Record UX decisions

## Outputs

- Decision or action summary
- Updated source-of-truth files when requested
- Next recommended LeanOS command

## Navigation

Start from `../README.md`, choose a role in `../roles/`, load required skills in `../skills/`, then use this playbook.
