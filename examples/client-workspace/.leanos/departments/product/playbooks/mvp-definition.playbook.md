# MVP Definition Playbook

## Purpose

Turn strategy into the smallest coherent validation scope.

## When to Use

Use this playbook when the active Product role needs a sequence of execution.

## Sequence

1. Read strategy files
2. Define MVP scope
3. Write user stories
4. Define acceptance criteria
5. List non-goals

## Outputs

- Decision or action summary
- Updated source-of-truth files when requested
- Next recommended LeanOS command

## Navigation

Start from `../README.md`, choose a role in `../roles/`, load required skills in `../skills/`, then use this playbook.
