# Issue to PR Playbook

## Purpose

Move from a scoped issue to a reviewable pull request.

## When to Use

Use this playbook when the active Engineering role needs a sequence of execution.

## Sequence

1. Read issue and MVP scope
2. Plan implementation
3. Change code
4. Update tests
5. Prepare PR

## Outputs

- Decision or action summary
- Updated source-of-truth files when requested
- Next recommended LeanOS command

## Navigation

Start from `../README.md`, choose a role in `../roles/`, load required skills in `../skills/`, then use this playbook.
