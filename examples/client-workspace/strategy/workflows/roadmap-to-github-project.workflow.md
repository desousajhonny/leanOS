# Roadmap To Github Project Workflow

## Purpose

Prepare roadmap, milestones and epics for GitHub Project sync.

## Required Areas

- roadmap
- product

## Availability

All required areas are active in this department.


## Sequence

1. Read roadmap and current cycle
2. Confirm product outcomes and priorities
3. Prepare milestones and epic drafts
4. Ask DevOps to validate GitHub project settings when needed
5. Produce payload and ask for confirmation before API execution

## Navigation

Use Strategy area READMEs for each step to preserve area-first ownership.
