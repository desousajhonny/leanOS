# Validation Cycle Planning

## Purpose

Plan the next coherent validation or delivery cycle.

## Area

`strategy/roadmap`

## Sequence

1. Read product strategy
2. Read validation risk
3. Review backlog
4. Choose current cycle
5. Record milestones

## Outputs

- Decision or action summary
- Updated source-of-truth files when requested
- Next recommended LeanOS command
